# 2026 청년다다름사업 지침 반영 초안

## 목적

이 문서는 `2026년 청년다다름사업 운영지침(사업비 예산편성 및 집행 지침)`을 현재
지출품의서/지출결의서/회계관리 앱에 반영하기 위한 1차 설계 초안이다.

이번 단계의 목표는 세 가지다.

1. 서식과 회계관리에서 반드시 받아야 할 필드를 정리한다.
2. DB와 화면이 어디까지 바뀌어야 하는지 검토 가능한 수준으로 구체화한다.
3. 다음 구현 단계에서 바로 사용할 수 있는 검증 규칙의 기준을 남긴다.

## 지침에서 직접 반영해야 하는 핵심 규칙

### 공통 증빙

- 공통 기본서류: `지출결의서(품의서)`, 필요시 `역량강화지원금 신청서`
- 세금계산서 건: 거래처 사업자등록증, 통장사본, 세금계산서
- 카드결제 건: 구매사이트 화면캡처, 거래명세서, 카드전표
- 청년 개인 계좌이체 건: 신분증, 통장사본, 이체내역서
- 계약성 지출: 계약서, 거래명세서 또는 견적서 추가

### 집행 통제

- 사업비는 사업비계좌와 연계된 법인카드/체크카드 또는 계좌이체만 인정
- 현금 지급은 불인정
- 사업계획서와 승인 예산 범위 내 집행만 인정
- 직접비와 간접비 간 전용 불가
- 세목 전용은 최초 편성예산 기준 누적 10% 미만만 자체 처리 가능
- 사업 종료 1개월 전에는 비목 간 전용 불가
- 환급 또는 공제 대상 부가가치세는 집행액에서 제외

### 회계관리

- 기관 일반회계와 분리된 사업비 전용 계좌 필요
- 개인 명의 계좌 불가
- 사업별 별도 원장 필요
- 증빙은 월별/비목별로 구분 편철
- 사업 종료 후 최소 3년 보관
- 정산 시 통장거래내역, 품의서, 지출결의서, 세금계산서, 거래명세서 등을 제출

### 항목별 별도 기준

- 회의비 식비: 1인 15,000원 한도
- 내부 직원만 참여한 회의는 불인정
- 회의비는 회의일시, 장소, 목적, 참석자 서명이 있는 회의록 필요
- 사업추진비: 운영인건비의 5% 이내, 1인 30,000원 상한
- 국내여비: 출장 목적, 일정, 산출내역, 결과보고 필요
- 해외출장: 집행 불가
- 강사료: 특강강사 200,000원 초과 시 협의 필요, 1일 3시간 초과분 50% 감액, 1일 800,000원 한도
- 제한업종 및 부적정 집행 유형은 시스템 경고 규칙으로 반영 필요

## 1차 구현 범위

이번 1차 작업에서는 실제 집행 로직을 전부 바꾸기보다, 아래의 기반을 먼저 만든다.

## 화면 변경안

### 1. 기관/사업 선택 레이어 추가

- 로그인 직후 또는 문서 작성 시 기관과 사업을 먼저 고르도록 변경
- 같은 코드베이스에서 여러 기관과 여러 사업을 분리 운영

### 2. 지출품의서 폼 확장

- 현재 단순 항목 입력에서 `예산구분 > 비목 > 세목` 흐름 추가
- `지급방법`, `거래처정보`, `VAT`, `전용 여부`, `승인 필요 여부` 입력 추가
- 품의 단계에서 필수 증빙 예정 항목을 체크리스트로 선택

### 3. 지출결의서 폼 확장

- 품의서 기반 자동 채움
- 실제 지급정보와 증빙 업로드 상태를 별도로 기록
- 카드/계좌이체/청년 실비 지급에 따라 다른 첨부 체크리스트 노출

### 4. 회계관리 화면 추가

- 사업별 예산표
- 비목/세목별 집행 현황
- 월별 편철 목록
- 정산 제출용 문서 묶음 상태

### 5. 관리자용 설정 화면 추가

- 기관별 템플릿
- 결재라인
- 제한업종 경고 키워드
- 기본 증빙 체크리스트

## 검증 규칙 초안

### 저장 차단

- 현금 지급 선택
- 사업 종료일 이후 지급
- 직접비/간접비 전용 입력
- 필수 증빙 누락 상태로 최종확정
- 회의비인데 회의록 정보 누락
- 출장비인데 출장정보 누락

### 경고

- 세목 전용 누적 10% 이상
- 사업 종료 1개월 전 비목 전용 시도
- 회의비 1인당 15,000원 초과
- 사업추진비 1인당 30,000원 초과
- 강사료 200,000원 초과 특강
- 비교견적 없이 고가 지출
- 필요수량 대비 과다 집행 가능성
- 제한업종 키워드 감지

## 검토 포인트

아래는 사용자가 보고 우선 결정해주면 좋은 항목이다.

1. `역량강화지원금 신청서`를 앱 안에서 직접 생성할지, 첨부만 받을지
2. `직접비/간접비`를 한 화면에서 함께 관리할지, 별도 메뉴로 나눌지
3. `기관별 별도 배포`를 유지할지, 하나의 앱에서 기관 전환으로 운영할지
4. `증빙 파일 업로드`를 먼저 붙일지, 텍스트 체크리스트부터 갈지
