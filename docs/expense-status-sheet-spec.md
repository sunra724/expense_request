# 사업비 지출현황 총괄표 컬럼안

## 목적

이 문서는 실제 집행 건을 월별/사업별로 정리하는 `사업비 지출현황 총괄표`의 최종 컬럼안을 정리한다.

앱의 개별 문서 입력 데이터와 엑셀 총괄표를 1:1에 가깝게 연결하는 것이 목표다.

## 운영 원칙

- `지원기관`은 사업비를 교부하는 기관을 뜻한다.
- `운영기관`은 실제 집행 및 증빙을 관리하는 기관을 뜻한다.
- 현재 소이랩 운영 기준에서 총괄표의 기본 지원기관은 `청년재단`으로 둔다.
- 총괄표는 `한 줄 = 한 집행 건` 기준으로 작성한다.

## 최종 권장 컬럼

1. `No`
2. `등록일(지출일)`
3. `지원기관`
4. `운영기관`
5. `사업명`
6. `예산구분`
7. `비목`
8. `세목`
9. `지급금액(세전)`
10. `부가세`
11. `원천징수액`
12. `실지급액`
13. `거래처/수령인`
14. `세부내용`
15. `결제방법`
16. `증빙서류`
17. `증빙상태`
18. `지출품의번호`
19. `지출결의번호`
20. `비고`

## 현재 앱 필드와 매핑

- `등록일(지출일)` -> `record_date`
- `지원기관` -> `organization_id`로 선택한 기관명
- `운영기관` -> `org_name` 또는 설정의 단체명
- `사업명` -> `project_name`
- `예산구분` -> `budget_scope`
- `비목` -> `budget_category`
- `세목` -> `budget_item`
- `지급금액(세전)` -> `supply_amount`
- `부가세` -> `vat_amount`
- `원천징수액` -> 별도 정산 필드 필요
- `실지급액` -> `eligible_amount`
- `거래처/수령인` -> `payee_company`, `payee_name`
- `세부내용` -> `items[].description` 요약 또는 메인 적요
- `결제방법` -> `payment_method`
- `증빙서류` -> `evidence_type` + `evidence_checklist`
- `증빙상태` -> `evidence_completion`
- `지출품의번호` -> 연결된 `proposal.doc_number`
- `지출결의번호` -> `doc_number`

## 추가 구현이 필요한 항목

아래 항목은 현재 앱에 1차로 없거나 약한 상태라서 다음 구현 단계에서 보완하면 좋다.

- `원천징수액`
- `운영기관`
- `증빙상태`의 표시값 표준화
- `거래처/수령인` 병합 출력 규칙
- 엑셀 다운로드 포맷

## 엑셀 출력 권장 규칙

- 날짜는 `YYYY-MM-DD`
- 금액은 숫자 저장, 표시형식만 통화로 적용
- `결제방법`, `증빙서류`, `증빙상태`, `예산구분`은 고정값 목록 사용
- `지출품의번호`, `지출결의번호`는 텍스트로 저장
- 월별 시트와 연간 총괄 시트를 함께 제공하면 정산에 유리
